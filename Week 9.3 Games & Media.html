Shooting The Donut The Player Creates

There was a problem with the Sling shot where after the first ball was shot then the 2nd ball that was spawned in will just be hanging and moving about the place which would effect the pulling and launching of the sling shot. All i had to do was in the script that spawned the Donuts i just had to reenable isKinematic after the Donut was spawned.

Next was getting the Donut that the player creates in the Sprinkles mini game into the Packing mini game. The way i wanted to do it at first was to create and save a prefab but when i did my research to see how to go about this i realised that i should go about it my way first so that i at least have the feature there and then later on i can convert it to the way i originally wanted it to be like.

The way i went about this is that when you press the Next level button on the Sprinkles mini game the Donut that was created will have a script added to it using the addComponent function that wont destroy the Donut object when it loads to the next scene.

When the Donut is loaded to the next scene which is the Packing Mini game there will be a DonutManager which is a script that will look for the Donut that was loaded into the scene. Once the DonutManager finds the Donut Object then it will change the size, position and rotation of the Donut to make it fit into the sling shot, then after that the script would destroy the components that are not needed e.g. the Icing Collision script.

Then after that the script will add all the components to the Donut that is needed for the Donut to be able to be launched from the Sling shot. Once the components have been added then some of them will need to be modified for example, the SpringJoint2D Component will need a value in the connectedBody variable and its frequency value will need to be changed, then finally we get rid of the DontDestroyOnLoad script since it has done its job.
