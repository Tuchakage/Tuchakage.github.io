Trying Out C++ On Unreal For The First Time!

Now its time for me to explain the struggle i had............ Since it was my first time working with C++ i decided the first script i should make should be the health script, which just includes the players health and a function called TakeDamage() that has a parameter and whatever number is in the parameter is how much damage will be done to the player, that was the easy part.

Then came trying to make to override the OnComponentBeginOverlap function in C++ in a new script i created for Weapons. The problem i kept having was that i kept getting an error message whenever i tried to override the OnComponentBeginOverlap function and the reason for that is because i was trying to do it to a class of type ActorComponent.

I spent the whole day doing different methods of trying to get this to work and doing a lot of research, eventually i went to bed. Then when i woke up the next day an idea came to my mind that i somehow didnt think off for the previous day and that was making a variable of type CollisionMesh in the class of type ActorComponent and then overriding with that variable. I got out of bed, went on my Computer, tried the idea and i finally had no error messages.

The next thing i had to do was assign the CollisionMesh to the variable in the editor at first i tried to do it in the BeginPlay() function but i was getting an error but all i had to do was put it in the ConstructionScript function which runs before the BeginPlay() function.

Although i spent a whole day trying to do this one thing i wouldnt say it was a waste of a day because from me struggling on this i now had a much better understanding of C++ on Unreal than before.
