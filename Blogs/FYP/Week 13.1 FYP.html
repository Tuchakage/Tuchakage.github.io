Week 13 Small But Many Improvements

Date Created: Apr 6, 2022

New Lobby!

Since the deadline is nearing and the core mechanics have been implemented it was finally time to add proper winning and losing conditions, but before that i created a lobby where the player will first spawn in when they start the game which will take them to the arena where the combat starts when they walk through a trigger, it will also be the place the player returns to when they lose all their health or when they beat all the waves.

The player now spawns in with the PlayerStart Actor (Basically a spawnpoint for the player) but when adding this the TraceGround Event of the VR Character Animation blueprint wasnt able to access the skeletal mesh straight away which would produce an error. To fix this problem an IsValid node was added just before the animation blueprint goes into the FootIK function to make sure that the reference to the skeletal mesh has been assigned.

In the Lobby i also made it so that the player can adjust the player height of the character model so there character will be standing when they are in the Arena but this feature doesn't really work that well because the scaling of the Lobby and the arena is different.

Small Improvements

I also noticed how when the weapon is just lying on the ground and the Enemies touch it, they will still take damage, to solve this problem inside of the weapons C++ script that deals with the damage i added another condition where damage will only be dealt if the variable "hasBeenGrabbed" is true. This hasBeenGrabbed variable will be set to true in any WeaponBP when the weapon is picked up by the player.

There was also another problem where if you stop grabbing with the 2nd hand of a 2 handed weapon then the weapon will stay stuck on the first even if you have left go off the trigger on the left, for this i simply separated the "AttachedActor" variable to be "AttachedActorL" and "AttachedActorR".
