Redoing Pickup For Both Hands

So i could pick up the weapons, but only with one hand. It was quite easy to solve this problem since I had already done grabbing for one of the hands I just had to make extra functions to do it for the other.

To make it so that the weapons know what hand they need to be attached to I first created sockets on both hands for both the Bow and dagger, then in the blueprints for the weapons when the pickup function is called than a switch case is there to determine what socket the weapon goes to.

The hand Enum variable is set in the Pickup interface function inside of either the GrabActorLeft() function or GrabActorRight() function of the VRCharacter BP, thanks to the blueprint interface it allows the blueprints to communicate to each other and that is how the VRCharacterBP is able to tell the weapons what hand the weapon should be attached to.

Changes To Dagger

Some minor changes were made to the dagger such as an actual mesh for the dagger was added and whilst play testing the game for myself i realised that whilst holding the dagger you were still able to teleport to it. To make sure that this can't happen the variable that tells the VRCharacter Blueprint that it is grabbing was used to create a branch that checks whether or not the player is grabbing onto something, if it is then the player cannot teleport to the dagger.
