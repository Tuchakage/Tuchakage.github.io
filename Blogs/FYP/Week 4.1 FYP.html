Date Created: Nov 24, 2021

Launching The Arrow

Before spawning in the Arrow i had to first make the ArrowBP, in this Blueprint i created a shoot function that when called the arrow will be launched, the speed of the arrow depends on the string charge variable in the BowBP and i created an onComponentBeginOverlap that when that hitbox touches something then the arrow will stop moving and can no longer collide with anything else.

The arrow is spawned when the player grabs hold of the string with their second hand and the arrow is attached to the string of the bow so it will follow the string. When the player lets go of the string in the shoot event in the BowBP the arrow will be detached from the string and the shoot event inside the arrow will be called which launches the arrow forward.

Problem With The Bow & Arrow & The Solution

I was having a problem where when i launched the arrows then the arrows would collide with the bow or motion controllers, the reason for this is because the Arrow spawns inside the bow. I was able to fix this problem by after attaching the Arrow to the string i would then set the X location of the arrow so the back of the arrow is actually touching the string and i also changed the limit for how far back you can pull the string back.
