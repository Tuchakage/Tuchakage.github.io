Week 4 - Enemy Taking Damage

Date Created: Nov 26, 2021

Thinking About How To Go About VR Combat

After finishing the Bow & Arrow it was time to start the VR Combat section of my project but i wasnt sure how i should go about it, thats when i wrote down the tasks i should do on a sticky note (See "VR Combat Tasks" Image).

After making a checklist of Sticky Note i finally had a good understanding of what i should do, so i went onto Unreal and created a static mesh actor for my Enemy and set the Mesh to a cube for now.

Making The Enemy Take Damage

I wanted to first see if i could make the Enemy take damage using the C++ Health script i made earlier in the project. So i added the Health component to the EnemyTestBP and in the Weapon Component script i changed the variable from "player" to "enemy" and then played the game.

When the weapon would fall and collide with the Enemy cube then Unreal would crash, i found out later that this was happening because the Weapon script was trying to access the "enemy" variables health component but the "enemy" variable was set to null.

To fix this problem i made the rest of the OnOverlapBegin function inside of the Weapon script only be called if the "OtherActor" parameter has the tag of "Enemy" and changed the "enemy" variable to call the TakeDamage function and replaced it with the "OtherActor" parameter instead.
