Creating Network Synced Players

This week was the final week before submission and a lot of good progress was made! One of them was creating the Player that works with multiplayer as the other programmer has been focusing on the player mechanics, i decided to get started with making it be usable in multiplayer since i had finished all my tasks. Since during this whole project i have been working on the system side of the game and previously i mostly made games in the Unreal Engine it was finally time for me to learn about the new Unitys Input System!

For the Pet character i created a new Script called Pet Controller that would handle just basic movement, jumping and light attacks, to make it multiplayer compatible so that all the players can control their own character at the top of the FixedUpdate() function all i needed to do was add the code “if(!photonView.IsMine) return;” which means if i don’t control this photon Instantiated object then don’t run the rest of the code. The Wizard uses a different script called “FloatingCharacter” created by the other programmer but i also added code to that to make it usable online.

Unitys Input System

The Unity Input system works similar to Unreals Enhanced Input System which allows for easy input management so that an action can be performed by different inputs from different controllers e.g. WSAD keys if the player is using a keyboard but if the player wanted to play with a controller moving with the left stick then they can do that without programmers having to modify the code.

I created action maps for Pets and Wizard which essentially manages the control scheme depending on the role that was chosen , this isseparate since they both have different abilities. For action maps you need to subscribe them to a function in your code for example Pet.Jump.started is subscribed to the Jump( Function and since i chose the “started” property, whenever the button to jump is pressed the Jump() Function is called, this is subscribed when the Player is spawned in. There is also code to unsubscribe the function from the action map whenever the player is destroyed, unsubscribing action maps can be helpful if something happens in the game where the same controls need to be used differently.
