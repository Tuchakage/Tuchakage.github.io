Players seeing how many Rounds they won

Not only could i not see how many rounds teams have won unless i looked in the console, players also had no idea how many rounds they won unless they kept count, because of this i decided to create a function called UpdateRoundWonUI() which is called at the start of the level being loaded as long as the key that holds the value of how many games each team has won has already been set.

In this function depending on the team that is being checked the code will look for the spawnLocation of where the round won indicator should be and it will then check the current rooms custom properties to check how many rounds that team has won and will spawn in the amount of indicators accordingly. The indicators are spawned in order thanks to the Horizontal Layout Group Component.

Login System

Registering

When registering i also learned about salt and has which prevents hackers breaking through encrypted passwords, to actually create a hash we need to first create a salt and we encrypt the salt and the password by using the crypt() function.

Logging In

For logging in we simply check if the username inputted is even part of the database and for checking the password we get the salt that is already in the database and with the inputted password we will use the crypt() f unction and if that results in being the same as the hash that is already stored in the database then the check was successful and the player is logged in.
