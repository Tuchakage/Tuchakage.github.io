Week 7 - Juicy Game Ideas

Date Created: Nov 22, 2021

Feedback From A Friend

I sent one of my friends who were interested in what game i was making a link to the game on the google play store and a google form sheet to give us some feedback. The feedback we received from his was very good, he mentioned how for the sprinkles mini game he had no idea what to do until i told him so that tells us that we have to add instructions to how to play each mini game.

He also mentioned how we need to make it so the game is beatable which we will obviously will do i just had to explain to him that it was a prototype showing off the core mechanics. He also gave us a few ideas one of them being to increase the conveyor belt speed as you put in more items into the correct container to make the game more challenging which i think is a good idea. The other idea was a whole new mini game which is Tetris but with Donuts instead.

Responding To Feedback

The first thing i did was increase the Conveyor Belt speed from what our Lecturer Hope told us last week about how she feels like the Conveyor belt moves way too slow even for a beginner level. I implemented a feature where the bowls will change sprite for when they are half full and another sprite for when they are full.

I did this in the **Container Script** which is where all the collisions for the containers are stored inside, i made it so first whenever an item is put into the correct container then a variable (depending on the type of item) will be incremented by 1. I then created an if statement that everytime an item is put into the correct container then it will check if it is half full or full and then from there it will change the sprite which are stored in a separate variable of type Sprite one for half full containers and one for full containers. This has only been done for the Measuring Cup (Milk Container) and the Yeast Bowl, i am currently waiting for the other container sprites to be done.

Group Discussion

Me, Shaafie and Hassan had a Discord where we discussed how the winning and losing condition would work. Hassan suggested that we should make it so that if the player get 3 items into the wrong container then they will lose the level, i thought that was a good idea until i realised that will make the Timer completely useless thats when Shaafie said that we could save that for an extremely hard level. We all agreed that you win the level by sucessfully filling up all the containers.

Finally we added another feature to make the game hard which is where if you put the wrong item in a container that has already been filled up a bit, then the amount of the item in that ingredient will be set back by 1.
